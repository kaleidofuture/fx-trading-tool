# デモトレード運用ガイド

## 運用フロー

```
シグナルモニター起動 → 通知を受信 → OANDA Webで手動発注 → trade_log.py で記録
```

### 1. シグナルモニターを起動

```bash
cd 02_開発/04_シグナル通知
python signal_monitor.py
```

4時間足の確定タイミング（UTC 00:05, 04:05, 08:05, 12:05, 16:05, 20:05）に自動チェック。
シグナルが出るとデスクトップ通知が届く。

### 2. シグナルに従って手動発注

OANDA Japan のデモ口座 Web 画面から発注する。
- 通知に表示された方向（LONG/SHORT）で発注
- SL（ストップロス）とTP（テイクプロフィット）を設定
- **取引数量は1000通貨（最小単位）から始める**

### 3. トレードを記録

```bash
cd 03_運用/01_デモトレード記録

# 新規トレード記録
python trade_log.py add

# オープンポジション確認
python trade_log.py list

# トレードクローズ
python trade_log.py close 1

# 全履歴
python trade_log.py history

# パフォーマンスレポート
python trade_log.py report
```

## 判断基準

### エントリー条件（シグナルモニターが検出）
- EMA(200) の上/下にいること（トレンド方向）
- EMA(20) と EMA(50) のクロス
- RSI が 30-70 の範囲内

### ポジションサイズ（1%ルール）
- デモ口座資金: 300万円
- 1トレードのリスク上限: 3万円（1%）
- 数量 = 3万円 ÷ (SLまでのpips × pip単価)

### 撤退ルール
- 累計ドローダウンが60万円（20%）を超えたら全停止
- 5連敗したら1週間休止して戦略を見直す
